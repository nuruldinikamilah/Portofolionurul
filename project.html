Exploring Document Retrieval Using Retrieval-Augmented Generation (RAG)

This project explores the fundamentals of Retrieval-Augmented Generation (RAG) by building a custom document retrieval pipeline from scratch. The system processes markdown-based study documents, splits them into optimized text chunks, converts each chunk into dense vector embeddings using the Hugging Face model `sentence-transformers/all-MiniLM-L6-v2`, and stores them efficiently using ChromaDB for fast semantic retrieval.

  • Document loader: Reads and extracts content from `.md` files placed in the `/docs` directory using LangChain’s `DirectoryLoader`.
  • Text Chunking: Utilizes `CharacterTextSplitter` to break documents into manageable text chunks to improve embedding and retrieval quality.
  • Embedding and Vector Storage: Leverages `sentence-transformers/all-MiniLM-L6-v2` to convert chunks into embeddings and stores them using ChromaDB with persistent storage.

Python (LangChain, ChromaDB, HuggingFace Embeddings), Sentence Transformers (all-MiniLM-L6-v2), Markdown document processing

Internet Content Classification System Using Multimodal LLM and Support Vector Machine with a Human-in-the-Loop Approach

The development of this parental control application uses two types of data, namely text and images from internet websites, utilizing a Multimodal Large Language Model (MLLM) approach to perform image captioning, so that the system can understand and provide descriptions of images that are potentially ambiguous, especially images that are difficult to classify based solely on pixel-level analysis.
The final text results are then classified using the Support Vector Machine (SVM) algorithm to determine whether the content contains negative elements or not. To improve the model's ability to adapt to new types of negative content that continue to evolve on the internet, the system implements Human-in-the-Loop (HITL). In this mechanism, parents are involved in providing annotations so that the new data can be used for periodic retraining of the model to make the system more accurate and relevant in the future.

  • Performed NLP preprocessing using regex, NLTK, tokenization, and TF-IDF.
  • Developed real-time inference API (Flask) to classify URL content (text + images).
  • Built model retraining loop with parent feedback (Human-in-the-Loop).

Tech: MLLM, SVM, HITL, KDD Process, Python(HuggingFace transformer, NLTK, Sastrawi, Regex, Scikit-Learn, Joblib, PIL, torch, matplotlib), PostgreeSQL

Tools: VSCode, DBeaver, Azure VM

Sales Performance Analysis For Pizza Retail Business

A data analytics capstone project from the Studi Independen Bersertifikat (SIB) program, focused on the Food & Beverage (F&B) industry. The project analyzed the sales performance of a pizza restaurant that had been operating for a year in 2015. The business experienced monthly sales fluctuations, and based on the insights, the goal was to increase the average sales growth to 8% in 2016 through data analysis.

  • Conducted EDA that revealed unstable performance, including sales declines in 58.3% of months and a negative growth trend of –7.3%.
  • Analyzed key KPIs (monthly revenue, AOV, order frequency, category performance) to identify root causes of declining sales.
  • Recommended improvements that supported the target of increasing next-year average growth to +8%, and visualized findings using Looker Studio dashboards.

Tools: Excel, Python, SQL, Bigquery, Looker Studio, HTML, CSS, JavaScript

Analysis of Flood Causes and Predictive Modeling of Flood Events in All Regions of DKI Jakarta

This data mining research aims to analyze the causes of flooding in Jakarta and develop a predictive model for flood occurrences in each region of DKI Jakarta in 2020 by examining rainfall levels and water height, in order to provide actionable recommendations for mitigation. The methodology used follows the KDD process, including data selection, data preprocessing and transformation, and data mining using the ARIMA Multivariate Regression algorithm.

Tools: Python(numpy, pandas, statistics, matplotlib, seaborn, plotly)
